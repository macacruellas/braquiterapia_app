📦 Descripción de Módulos

🛣️Routes (Rutas)

  • main_routes.py: Página de inicio
  • dvh_routes.py: Carga de DVH, cálculos EQD2
  • export_routes.py: Generación de informes

📄Parsers

  • eclipse_parser.py: Lee archivos .txt de Eclipse
  • oncentra_parser.py: Lee archivos .txt de Oncentra
  • Extracción de DVH, metadatos de pacientes

🧮Calculations

  • dosimetry.py: Cálculos físicos
  • EQD2, BED, dosis por fracción
  • Ecuaciones cuadráticas para HDR

🔧Utils

  • helpers.py: Conversiones, validaciones
  • roi_mapping.py: Mapeo ES/EN de ROIs
  • file_handlers.py: Excel, PDF, LibreOffice

🎨Templates & Static

  • base.html: Layout base con CSS/JS
  • styles.css: Diseño moderno, responsive
  • main.js: Interactividad del frontend

⚙️Config

  • settings.py: Configuración centralizada
  • Límites de dosis, constantes α/β
  • Valores por defecto del sistema

✨ Ventajas de esta Estructura

  • ✅ Modularidad: Cada módulo tiene una responsabilidad clara
  • ✅ Mantenibilidad: Fácil encontrar y modificar código
  • ✅ Escalabilidad: Agregar features sin tocar código existente
  • ✅ Testing: Cada módulo se puede testear independientemente
  • ✅ Colaboración: Múltiples personas pueden trabajar en paralelo
  • ✅ Separación de concerns: Lógica, presentación y datos separados
